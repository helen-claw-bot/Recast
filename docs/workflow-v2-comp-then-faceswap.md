# 剧照生成工作流 v2：构图先行 + 换脸

> 解决 AI 同时兼顾"构图动作"+"人物一致性"效果差的问题

## 核心思路

将剧照生成拆为两步，每步只解决一个问题：

### Step 1 — 构图生成（Gemini 文生图）

**目标：** 动作合理、环境一致、光线一致、构图准确
**不管：** 人脸是否像角色

审核标准：
- ✅ 人物动作符合剧本描述（手的位置、身体朝向、距离感）
- ✅ 环境细节一致（矮墙、梅酒杯、薄荷、灯串）
- ✅ 光线色温统一（暗蓝天光 + 琥珀暖点）
- ✅ 景别正确（特写/近景/中景）
- ✅ 视线方向正确（Chu 看右，Leo 看左）
- ✅ 左右手连续性（跨镜头不换手）
- ⚠️ 发型和服装尽量接近角色设定（减少 Step 2 违和感）

### Step 2 — 换脸（可灵角色参考 / 即梦角色一致性）

**目标：** 替换面部为固定角色参考图
**保持：** 构图、动作、环境、光线不变

工具选择：
- **可灵「角色参考 + 图片编辑」** — 推荐，控制力强
- **即梦「角色一致性」** — 备选，免费额度多
- **InsightFace / Reactor** — 开源方案，精度高但需要本地跑

注意事项：
- 换脸可能微调发际线、脖子、衣领区域
- Step 1 生图时发型要接近角色设定（楚然：黑长发散落；Leo：长发松散小揪揪）
- 侧脸/半遮挡镜头换脸效果可能不稳定，必要时手动修

### Step 3 — 图生视频（可灵）

- 用 Step 2 换脸后的图作为首帧
- 粘贴对应的 motion prompt（见 `storyboard/01-rooftop-kling-prompts.md`）

## Gemini 文生图已知坑（持续更新）

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 人物名触发错误形象 | "Chu Ran" 被当作已知人物 | 用 "Chu" 或 "Character A" |
| 对视眼神不聚焦 | 未指定视线方向 | 明确 "looking right/left" |
| 手性别错误 | 默认偏男性化 | 加 "young East Asian woman's hand, slender, feminine" |
| 指甲脏感 | 阴影渲染问题 | 加 "clean pink nail beds, no dirt" |
| 指甲过长 | "almond-shaped" 触发美甲 | 改用 "short trimmed natural nails" |
| 手臂角度不听话 | 16:9 横构图限制 | 不要写 "vertical"，改为只露手腕 |
| 倚靠空气 | 缺少依靠物描述 | 明确写墙/栏杆作为支撑 |
| 左右手翻转 | 跨镜头不一致 | prompt 里固定 "right/left hand" |
| 酒色错误 | "plum wine" → 紫色 | 改为 "umeshu, golden-amber" |
| 正面像证件照 | "facing camera" 太直白 | 改为 "three-quarters angle" |
